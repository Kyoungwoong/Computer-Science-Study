# 정대훈 Network Overall Feedback

## Interview

- ### Q) TCP/IP 4계층 모델에 대해서 설명해주세요


        A) 애플리케이션 → 트랜스포트 계층 → 네트워크 → 링크가 있다

    #### 정균
  > 대답하신 부분은 피드백 때 말씀드린 것 처럼 OSI 7계층의 이름을 4계층으로 바꾼 듯한 느낌이고 정확한 TCP/IP 4계층 모델의 명칭은 애플리케이션 -> 전송 -> 인터넷 -> 네트워크 액세스 계층입니다!!

    <br>

- ### Q) 애플리케이션 계층에는 무슨 프로토콜이 있나요?

        A) 대표적으로 HTTP나, HTTPS가 있습니다.

    #### 정균
  > 맞는 답변이긴 하나, HTTP 말고도 FTP나 SMTP 같은 것도 말씀해주시고, 답변을 할 때, 이게 **무슨** 프로토콜인지 간단하게 설명을 해주시면 좋을 것 같습니다.

  > 애플리케이션 계층에는 웹 브라우저와 웹 서버 간에 데이터를 주고받는데 사용되는 HTTP 또는 HTTPS, 그리고 컴퓨터와 컴퓨터 간에 파일을 주고받을 때 사용되는 FTP, 또 이메일 메시지를 보낼 때 사용되는 SMTP 등이 있습니다.

  > 이런식으로 말씀해주시는게 더 전문성 있는 답변으로 보입니다.

    <br>

- ### Q) 그러면 HTTP랑 HTTPS의 차이점에 대해서 설명해주시겠어요?

        A) HTTPS가 HTTP보다 보안이 더 뛰어나고, 그 이유는 데이터를 보내는 과정에 있어서 SSL이나 TSL 암호화를 하기 때문입니다.

    #### 정균
  > TSL이 아니구 TLS입니다!! 용어 정리 잘 해주셔야 할 것 같아요 그리고 강사님께서 꼬리질문으로 어떻게 암호화를 하냐고 여쭤보셨는데 이 부분에 대해선 아래에 정리를 해두겠습니다!!

    <br>

- ### Q) 암호화는 어떤 방식으로 할까요?

        A) 잘 모르겠습니다.

    #### 정균
  > 클라이언트가 서버에 접속하면, 서버는 SSL/TLS 인증서를 제공하며 공개키를 보낸다.
  > 클라이언트는 이 공개키를 이용해 앞으로 사용할 대칭키를 암호화하여 서버에 전달한다.
  > 서버는 자신의 개인키로 대칭키를 복호화하고, 이후부터는 이 대칭키를 사용해 통신 데이터를 암호화/복호화하며 통신한다

- ### Q) HTTP 메서드와 이것이 하는 역할에 대해서 설명해보세요

        A) 클라이언트가 데이터를 요청하는 POST, 쿼리스트링을 포함해서 요청하는 GET, 그 외에는 DELETE, PUT, PATCH가 있습니다.

    #### 정균
  > POST는 데이터를 요청할 때 사용을 할 수도 있겠지만, 서버에 새로운 자원을 생성하는게 주 목적입니다. 이 부분을 알아두셔야 할 것 같고, 멱등성에 대해서도 공부를 해보시면 좋을 것 같습니다!!

    <br>

- ### Q) 쿼리스트링이 뭔가요?

        A) URL에 붙여서 보내는게 쿼리스트링이라고 알고 있습니다.

    #### 정균
  > 답변 잘 하셨으나, 조금 더 부연 설명이 필요해보입니다!!
  > 쿼리스트링은 클라이언트가 서버에 추가적인 정보를 전달하기 위해 URL의 끝에 덧붙이는 문자열이고 주로 GET 메서드와 함께 사용되어 서버에서 특정 리소스(자원)를 조회하거나 필터링할 때 필요한 데이터를 담습니다.

- ### Q) PUT과 POST의 차이점에 대해서 설명해주세요

        A) 잘 모르겠습니다.

    #### 정균

- ### Q) IP 주소와 MAC 주소를 연결하는 프로토콜은 무엇인가요?

        A) IP주소로 MAC주소를 연결할 때 사용되는 프로토콜은 ARP입니다.

    <br>

- ### Q) ARP은 어떻게 동작하나요?

      A) 잘 모르겠습니다.

    #### 정균
    > ARP Request와 ARP응답, 그리고 ARP 캐시 과정 정도로 답변하시면 될 것 같습니다!!<br><br>ARP 요청<br>
        호스트 A가 같은 네트워크에 있는 호스트 B에게 데이터를 보내려고 할때, IP주소는 알지만 MAC주소는 모른다.<br>
        이때, A는 B의 IP주소(192.168.1.10)를 가진 호스트의 MAC주소가 뭐에요? 라는 메시지를 네트워크 전체에 브로드캐스트로 전송한다.
        <br><br>ARP 응답<br>
        네트워크의 모든 장치가 이 브로드캐스트 메시지를 받는다. B는 메시지의 IP 주소가 자신과 일치함을 확인하고, A에게 자신의 MAC주소를 담은 ARP응답 메시지를 보낸다. 이 응답은 유니캐스트로 전송된다.
        <br><br>ARP 캐시<br>
        A는 B로부터 받은 MAC주소를 ARP 캐시라는 임시 저장소에 저장하고, B와 통신할 때, 이 ARP캐시를 먼저 확인하여, ARP 요청을 다시 보낼 필요 없이 바로 MAC 주소를 사용해서 통신한다.

    <br>

        A) 송신자와 수신자 사이에서 상대방의 윈도우 사이즈를 알아가면서 전송하는 방식

    #### 정균
  > 답변은 Flow Control의 개념에 가깝고, Congestion Control은 네트워크 자체의 혼잡을 막는데 초점을 둬서 혼잡 제어는 네트워크가 과부하 상태(Congestion)에 빠지는 것을 방지하거나, 이미 발생한 혼잡을 완화하여 네트워크의 효율적인 자원 사용을 유도하는 메커니즘입니다!!
